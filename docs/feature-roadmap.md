## VCam2D Components

### Positioning Components

Components that control how the camera moves and zooms.

- [X] `VCam2DConfinementComponent`
	- Confines the camera to a rectangular area
- [X] `VCam2DFramingComponent`
- [ ] `VCam2DGroupFramingComponent`
	- Positions the camera at the mid-point between a group of specified nodes or between all nodes of a certain node
	group, while controling the camera zoom so that all such nodes are visible. You can specify screen-space margins
	to prevent the targets from being too close to the camera edges; and world-space margins to prevent undesired
	excessive zoom.
- [ ] `VCam2DPathConstraintComponent`
	- Constrains the camera position to a `Path2D` node.
- [ ] `VCam2DPlayerControlComponent`
	- Reads directional input from the player and let them control the camera position and/or zoom. You can determine
	how far it can see and whether the camera resets to its original position or not when the player releases the
	directional input. You can combine it with `VCam2DConfinementComponent` to set boundaries to the player control.
- [ ] `VCam2DShakeComponent`
	- Shake the camera's offset and/or zoom using noise textures
- [ ] `VCam2DRotationComponent`
	- Adds rotation to the camera.

### Prioritization Components

Components that automatically calculate and add priority to the camera based on various factors.

- [X] `VCam2DObjectDetectionComponent`
	- Connects to an `Area2D` node and adds priority based on the number of detected objects in the area.

### Image Components

Components that change the image produced by the camera.

- [ ] `VCam2DShaderComponent`
	- Applies a post-processing shader to the image generated by this camera.
- [ ] `VCam2DTextureComponent`
	- Add a static or animated sprite on top of the camera output or behind the scene.

### Transition Components

Components that control how one camera cuts to another.

- [ ] `VCam2DFadeComponent`
	- Fades the image to black and back when transitioning from another camera to this one
- [ ] `VCam2DTransitionComponent`
	- Apply a texture-based transition when transitioning from another camera to this one (i.e. apply a texture-based
	erosion/dissolve of one camera image over the second)
- [ ] `VCam2DTransitionUVComponent`
	- Apply a UV-texture-based transition when transitioning from another camera to this one
- [ ] `VCam2DTweenComponent`

### Aim Components
- [ ] `VCam3DFreeLookComponent`
	- Reads input from the player and let them control the camera direction.
	- > Cinemachine similar: https://docs.unity3d.com/Packages/com.unity.cinemachine@3.1/manual/CinemachineFreeLookModifier.html
- [ ] `VCam3DLookAtGroupComponent`
	- Looks at the mid-point position between a set of specified nodes or between all nodes in a specified node group.
	- > Cinemachine similar: https://docs.unity3d.com/Packages/com.unity.cinemachine@3.1/manual/CinemachineTargetGroup.html
- [X] `VCam3DLookAtTargetComponent`
	- Looks at a specified target node, with an optional offset.
- [X] `VCam3DMimicRotationComponent`
	- Sets the camera rotation based on a target node's rotation, with some optional offset. Useful for first-person
	and third-person cameras.

### Prioritization Components

Each priority controller should have a blend mode (either Add, Sub, or Override)

- [ ] Animation-Based Prioritization Component
	- Sets the camera priority based on which animation is playing.
	- > Cinemachine similar: https://docs.unity3d.com/Packages/com.unity.cinemachine@3.1/manual/CinemachineStateDrivenCamera.html
- [X] Framing Priority Component
- [X] Line-Of-Sight Priority Component
- [ ] Node-Count Priority Component
	- Controls priority by counting the number of nodes on the scene with a specific group
- [ ] Player Input Priority Component
	- Adds priority when the player enters camera-control inputs; and reduce priority later after a delay
- [X] Proximity Priority Component
	- Controls priority by multiplying the distance between two targets by a set factor and base values
- [ ] Velocity-Based Priority Component
	- This controller compares the camera's position between this and the previous frame to set priority
